introduction

breve is a simulation environment which allows users to define the behaviors and interactions of different types of agents in a simulated 3D world and observe the emergent behaviors. breve is conceptually similar to packages such as Swarm and StarLogo, but simulates objects in a continuous 3D world with continuous time. breve thus allows simulation involving realistic collision detection and physics.

Agent behaviors are written in a simple object oriented language called "steve". Users define an agent's behavior by specifying how the agent interacts with the world at each timestep and how the agent interacts when it collides with other agents.

breve includes full collision detection and physical simulation of rigid bodies. See the Walker and Gravity simulations for examples.

For a more detailed introduction of the breve environment, refer to the introductory paper at: http://www.spiderland.org/breve.

This release is version 2.0. Many changes have been made from release 1.9, so please read the Changes file to find out what's new.

getting started

The best way to get started and learn about programming in steve is to examine the demos from the demos menu and watch them run. The most simple demo showing how objects move around the world are RandomWalker. A fun demo showing how collisions can be handled is the DLA demo which shows a decentralized model of fractal growth. The more complicated Swarm demos show different types of "boids" models of flocking birds. The Gravity demo is a simple simulation using physical simulation features, while the Walker and SuperWalker demos is a more complicated example in which a simulated creature learns how to walk.

To run a demo, simply select the demo from the "Demo" menu, and push the play button. Make changes in the code window and restart the demo to see how your changes affect the simulation.

After trying the demos, you can consult the breve documentation to learn more about writing simulations in breve.

known bugs

  • Fullscreen mode (OS X) does not currently work on all machines with all monitor settings. Lowering resolution or the number of colors may help.

feedback

If you have any cool or useful simulations or classes that should be included with future releases, mail them as well.

If you would like to join the breve users mailing list, you can subscribe at http://lists.spiderland.org/mailman/listinfo/breve.
